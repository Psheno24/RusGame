## 3. Работы и заработок

### 3.1. Общие правила зарплаты

Базовые выплаты задаются для Омска в `data/jobSalaries.json`, затем умножаются на **городской коэффициент**.

#### Городские коэффициенты зарплат

| Город | × | Город | × |
|-------|---|-------|---|
| Омск | 1 | Новосибирск | 1.8 |
| Волгоград | 1.05 | Екатеринбург | 2 |
| Воронеж | 1.1 | Казань | 2.3 |
| Пермь | 1.15 | СПб | 3 |
| Красноярск | 1.2 | **Москва** | **4** |
| Уфа | 1.25 | | |
| Ростов | 1.3 | | |
| Самара | 1.35 | | |
| Челябинск | 1.4 | | |
| Н. Новгород | 1.5 | | |
| Краснодар | 1.65 | | |

#### Формула (фиксированные смены)

```
база_город = round(база_омск × городской_множитель)
разброс = round(база_город × 5%)
выплата = random(база − разброс, база + разброс) × множитель_навыков
```

#### Множитель навыков (`skillPayoutMultiplier`)

- Эталон: уровень **25**
- Минимум: **×0.7** (навыки ≈ 0)
- Максимум: **×1.5** (навыки ≈ 50+)

| Работа | Веса навыков |
|--------|--------------|
| Курьер | стойкость 100% |
| Таксист | вождение 60%, общение 20%, стойкость 20% |
| Кассир | общение 100% |
| Охранник | дисциплина 80%, здоровье 20% |

#### Прочее

- **Множитель времени суток**: в шаблонах не задан → **×1**
- **Множитель виталов**: **×1** (энергия/настроение на зарплату не влияют)
- **Репутация**: за смену `cooldown` → **+2** (макс. 1000)
- **Навыки**: каждые **10** смен → **+1** к связанному навыку (макс. 100)
- **Снижение КД от авто**: `КД_факт = КД_база × (1 − снижение%)`, скорость 20 → 5%, 100 → 25%

### 3.5. Таксист

Отдельная экономика — линия заказов, не фиксированная смена.

#### Требования

- Права категории B
- Телефон + тариф «Полный безлимит»
- Свой автомобиль или аренда `car-economy` (Lada Granta)

#### Тайминги

| Параметр | Значение |
|----------|----------|
| Обновление пула заказов | **5 минут** |
| Заказов в пуле | **4** |
| Авто-сход с линии при бездействии | **1 час** |

#### Выплата за поездку

Почасовая ставка (Омск, тариф «Эконом»):

```
5 мин  → 8 000 ₽/ч
30 мин → 5 000 ₽/ч
45 мин → 4 000 ₽/ч
(линейная интерполяция)

Выплата = round(почасовая_ставка × минуты / 60)
```

Множители: городской коэффициент × тариф заказа × навыки. Минимум **150 ₽**.

#### Множители тарифов

| Тариф | × |
|-------|---|
| Эконом | 1.0 |
| Комфорт | 1.25 |
| Комфорт+ | 1.45 |
| Бизнес | 1.8 |
| Премиум | 2.2 |

#### Класс авто (по комфорту)

| Комфорт | Класс |
|---------|-------|
| < 40 | Эконом |
| 40–59 | Комфорт |
| 60–79 | Комфорт+ |
| 80–89 | Бизнес |
| 90+ | Премиум |

#### Тарифы по городам

- Большинство: эконом, комфорт, комфорт+
- Екб: + бизнес
- Казань, СПб, Москва: + бизнес, премиум

#### Наличные (38% заказов)

| Событие | Шанс | Эффект |
|---------|------|--------|
| Не заплатил | 1.2% (+0.6%/звезда ниже 4.5) | Выплата **50%**, настроение −2 |
| Частичная оплата | 3.5% (+1%/звезда) | Выплата **60%**, настроение −1 |

#### Рейтинг таксиста

- Старт: **4.85** (3.5–5.0)
- Отклонение заказа: **−0.02**
- Хорошая поездка (≥4.5): **+0.01** (35% шанс)
- Плохая (<3.5): **−0.03** (40% шанс)

> Параметры расхода энергии в `taxiConfig.json` (8/6/4) **пока не применяются** в коде.

### 4.2. Покупка недвижимости

Каталог: `data/housingProperties.json`.

#### Формула дохода от сдачи

```
Гросс-аренда/мес = round(цена / 60)
Расходы% = престиж_расходы + тип_расходы

Престиж: 0–20→15%, 21–40→20%, 41–60→25%, 61–80→30%, 81+→35%

Чистый/мес = гросс − round(гросс × расходы% / 100)
Чистый/нед = round(чистый/мес / 4)
Доход за период = round(чистый/нед × periodMs / 7дней)
```

#### Примеры (Омск)

| Объект | Цена | Чистый/мес | Чистый/нед | За 30 дней |
|--------|------|------------|------------|------------|
| Студия | 500 000 ₽ | 7 000 ₽ | 1 750 ₽ | 7 500 ₽ |
| Пентхаус 30 млн | 30 000 000 ₽ | 275 000 ₽ | 68 750 ₽ | ~295 000 ₽ |

Москва пентхаус: **2.1 млрд ₽**, чистый **19.25 млн/мес**, **4.812 млн/нед**.

#### Механика сдачи

- Период: **30 дней**, выплаты **еженедельно**
- После окончания: **80%** шанс продлить; иначе пауза (+5% шанс/день)
- При продаже с активной сдачей — возврат жильцам неполученной части

#### Перепродажа

| Режим | Ставка |
|-------|--------|
| Trade-in (< 30 дней) | 90% |
| Trade-in (≥ 30 дней) | 120% |
| Продажа | 60% |

## 8. Путешествия

По расстоянию на карте (`travelCalc.ts`).

**Поезд:**

- Короткие (< 180): `1200 + расстояние × 11.11` ₽
- Длинные: `−1088 + расстояние × 18` ₽
- Максимум: **10 000 ₽**

**Самолёт** (не везде):

- `3500 + t × 12000` ₽, макс. **15 500 ₽**

## 10. Не влияет на деньги (пока)

- Энергия, настроение, здоровье — не штрафуют зарплату
- Голод — не участвует в экономике
- Репутация — только +2 за смену
- Энергия такси в конфиге — не реализована в логике поездок
